סיכום למבחן: פיתוח יישומי אינטרנט

מדריך למידה אינטראקטיבי - שאלות, תשובות והסברים ממוקדים

1. שפות צד לקוח: HTML, HTML5 & CSS3

HTML & HTML5: שפת סימון (Markup Language) המשמשת לבניית שלד עמוד האינטרנט. מורכבת מאלמנטים (כמו DIV שהוא בלוק קונטיינר) ותכונות (Attributes - מידע נוסף בתוך התגית). HTML5 הוסיפה משמעות סמנטית (כמו header, nav) ותמיכה מובנית בווידאו ואחסון מקומי.

CSS3 ועיצוב רספונסיבי: עיצוב רספונסיבי מבוסס על Media Queries ויחידות יחסיות שמתאימות את האתר לגודל המסך. ספציפיות ב-CSS קובעת איזה עיצוב יוחל במקרה של התנגשות: ID חזק מ-Class, Inline Style עוקף את שניהם, ו-!important עוקף הכל.
שאלת תרגול: מה נכון בנוגע לאלמנט DIV?
הסבר: DIV הוא "קונטיינר" גנרי. הוא תופס את כל רוחב השורה (Block) ואין לו משמעות סמנטית בפני עצמו (בניגוד לתגיות HTML5 כמו nav או header).
שאלת תרגול: כיצד ניתן לממש עיצוב רספונסיבי (Responsive Design)?
הסבר: עיצוב רספונסיבי מתבסס על Media Queries ב-CSS, המאפשרים לשנות חוקי עיצוב בהתאם לרוחב המסך הנוכחי.

2. לוגיקה וממשק: JavaScript & DOM

JavaScript: שפת התכנות של הדפדפן, מאפשרת אינטראקטיביות. משתנים ב-JS: `var` (ישן, טווח פונקציה), `let` (מודרני, ניתן לשינוי, טווח בלוק), `const` (קבוע). פונקציות ב-JS יכולות לעבור כפרמטרים לפונקציות אחרות (Callbacks).

DOM (Document Object Model): הייצוג המבני של ה-HTML כ"עץ" בזיכרון, דרכו JS יכולה לשנות, למחוק ולהוסיף אלמנטים בזמן אמת.

Fetch: מתודה המשמשת לביצוע קריאות שרת אסינכרוניות (AJAX) ומחזירה Promise.
שאלת תרגול: מה נכון בנוגע ל-DOM?
הסבר: ה-DOM הוא המודל (עץ היררכי) שהדפדפן בונה מקוד ה-HTML כדי ששפות תסריט (כמו JS) יוכלו לגשת אליו.
שאלת תרגול: מה תפקיד המתודה fetch ב-JavaScript?
הסבר: מתודת fetch מחליפה את ה-XMLHttpRequest הישן ומשמשת לתקשורת אסינכרונית מול ה-Backend.

3. ניהול מצב ונתונים: Cookies, Session & JSON

Cookies לעומת Session: עוגיות (Cookies) נשמרות אצל *הלקוח* (בדפדפן), יש להן הגבלת גודל, והן נשלחות אוטומטית לשרת. לא מומלץ לשמור בהן מידע רגיש. Session נשמר בצד *השרת* והוא בטוח הרבה יותר. השרת שומר לרוב רק "מזהה ייחודי" (Session ID) אצל הלקוח כדי לזהות אותו.

JSON: פורמט טקסטואלי קל-משקל להעברת נתונים בין שרת ללקוח. בנוי מזוגות של מפתח-ערך, כאשר המפתחות חייבים להיות מוקפים במירכאות כפולות.
שאלת תרגול: מה ההבדל העיקרי בין Cookies ל-Session?
הסבר: השרת משתמש ב-Session כדי לשמור נתונים מאובטחים אצלו, בעוד Cookies הם קבצי טקסט אצל הלקוח.
שאלת תרגול: מה נכון לגבי JSON?
הסבר: JSON (JavaScript Object Notation) הוא הפורמט הסטנדרטי להעברת מידע ברשת, המפתחות שלו תמיד דורשים מירכאות כפולות.

4. צד שרת וארכיטקטורה: Node.js & Backend

Node.js: אינה שפת תכנות אלא סביבת ריצה המאפשרת להריץ JS על השרת. היא עובדת בצורה של Single Thread מבוסס אירועים (Event-driven), ולכן היא מעולה בפעולות I/O אסינכרוניות (כמו גישה למסד הנתונים).

Backend & Web Apps: צד השרת (Backend) מטפל בלוגיקה העסקית, אימות נתונים ותקשורת מול בסיס הנתונים (DB). אפליקציית Web מבוססת על מודל לקוח-שרת ומתקשרת באמצעות פרוטוקול HTTP.
שאלת תרגול: מה נכון בנוגע ל- Node.js?
הסבר: Node.js לוקחת את מנוע ה-V8 של גוגל ומאפשרת להריץ בעזרתו JS בשרת. כדי לא להיתקע (Single Thread), היא משתמשת בתקשורת אסינכרונית.
שאלת תרגול: מה נכון בנוגע לשימוש בבסיסי נתונים ב- Node.js?
הסבר: מאחר ו-Node.js פועל בחוט ביצוע יחיד, כל פנייה החוצה (כמו למסד נתונים) חייבת להיות אסינכרונית כדי לא לעצור את שאר המערכת.
נוצר במיוחד כדי לעזור לך לעבור את המבחן. בהצלחה!
